# Product Requirements Document (PRD)
# Sistem Informasi Pemesanan dan Manajemen Villa (Versi 3.0 - Final)

## 1. Ringkasan Eksekutif & Filosofi Sistem
Sistem Informasi Pemesanan Villa ini dirancang dengan filosofi **100% Cashless dan Validasi Terpusat**. Sistem ini menyelesaikan masalah operasional lapangan (kebocoran dana oleh pegawai) dengan cara mewajibkan semua pembayaran ditransfer langsung ke rekening PT/CV, dan HANYA pimpinan (Super Admin) yang memiliki hak untuk memvalidasi uang masuk di dalam sistem. 

## 2. Tech Stack & Lingkungan Pengembangan
* **Backend Framework:** Laravel 13 (Gunakan struktur modular dan Service Pattern).
* **Database:** MySQL (Relasional dengan Foreign Keys).
* **Frontend:** Blade Template Engine + Tailwind CSS + Alpine.js.
* **Target Deployment:** Shared Hosting (cPanel). File *core* Laravel wajib diletakkan di luar `public_html` demi keamanan.
* **Integrasi Eksternal:** Telegram Bot API (Notifikasi & Laporan Berkala).

## 3. Hak Akses (Role-Based Access Control)
Sistem memiliki 2 (dua) level otorisasi yang sangat kaku:

### A. Super Admin (CEO / Manajer)
* **Validasi Keuangan (MUTLAK):** Satu-satunya role yang bisa mengubah `payment_status` dari *Pending* menjadi *DP* atau *Lunas*.
* **Master Data & Harga:** Mengatur harga kamar, membuat Voucher Diskon (untuk negosiasi harga), menambah menu makanan/ekstra.
* **Laporan & Analitik:** Memiliki akses ke Dashboard Analitik, Laporan Keuangan, dan Riwayat Pembatalan.

### B. Admin Pegawai (Resepsionis)
* **Operasional (Read-Only pada Keuangan):** Hanya bisa melihat jadwal tamu yang *Check-in/Check-out* hari ini.
* **Manajemen Status Tamu:** Bisa mengubah status tamu (misal dari *Booked* ke *In-House*, atau ke *Completed*).
* **Input Pesanan Tambahan:** Bisa menginput pesanan makanan/ekstra ke tagihan tamu, **TETAPI** tidak bisa memvalidasi pembayarannya.
* **Restriksi:** DILARANG KERAS memiliki akses mengubah harga, mengubah status bayar, atau menghapus data transaksi.

## 4. Alur Kerja Sistem (System Workflow & User Journey)

### Fase 1: Booking & Checkout (Publik / Tamu)
1. **Pencarian:** Tamu memilih tanggal *Check-in* (14:00) dan *Check-out* (12:00).
2. **Validasi Overlap (Penting):** Sistem mengecek DB. Jika ada kamar dengan `payment_status` = (DP / Lunas) pada rentang tanggal tersebut, kamar disembunyikan.
3. **Form Pemesanan & Analitik:** Tamu mengisi data diri dan memilih *dropdown* "Tahu Villa ini dari mana?" (Instagram/Google/Teman/dll) untuk data *marketing*.
4. **Voucher & Tagihan:** Tamu bisa memasukkan Voucher Diskon. Sistem menghitung Total Tagihan dan Minimal DP (persentase dinamis dari *Settings*).
5. **Pembayaran:** Tamu diarahkan untuk transfer ke Rekening Bank (Mandiri, BNI, BCA perusahaan) dan menekan tombol konfirmasi via WhatsApp.

### Fase 2: Validasi Terpusat (Telegram & Super Admin)
1. **Real-time Alert:** Saat tamu menekan konfirmasi, Telegram Bot menembak pesan ke Grup Super Admin: *"🚨 Booking Baru/Pesanan Tambahan #VLA-001 dari [Nama]. Silakan cek mutasi."*
2. **Eksekusi:** Super Admin mengecek mutasi di *mobile banking*, login ke web, dan menekan tombol **"Terima Pembayaran (DP/Lunas)"**. Jadwal kamar resmi terkunci.

### Fase 3: In-Stay (Menginap) & Pemesanan Tambahan
1. **Running Tab:** Tamu yang sudah menginap bisa memesan *Extra Bed* atau Makanan. 
2. **Proses:** Admin menginput pesanan ke sistem -> Tamu wajib transfer -> Bot Telegram memberitahu Super Admin -> Super Admin klik Lunas -> Dapur memproses makanan. Total tagihan ini tergabung dalam satu Invoice/Kwitansi akhir.

### Fase 4: Checkout & Housekeeping
1. **Late Checkout & Denda:** Saat tamu *checkout*, Admin bisa menambahkan biaya manual (opsional) jika tamu telat keluar. Tamu melunasi sisa tagihan via transfer.
2. **Status Kamar (Buffer):** Setelah tamu *checkout*, status kamar TIDAK langsung *Available*, melainkan **Cleaning / Pembersihan**. Setelah petugas selesai membersihkan, Admin menekan tombol "Kamar Siap", baru statusnya menjadi *Available*.

## 5. Fitur Dashboard & Laporan (Analitik Bisnis)
Dashboard Super Admin harus menampilkan metrik berikut:
* **Widget Utama:** Total Pendapatan Kotor bulan ini, Persentase Okupansi, dan Sisa Tagihan yang belum lunas.
* **Grafik (Charts):** *Line chart* tren pendapatan harian, dan *Pie chart* Sumber Referensi (Acquisition Channel).
* **Laporan Excel/PDF:** Ekspor data laporan keuangan yang bisa difilter berdasarkan bulan/tahun.
* **Telegram Auto-Report:** *Cron Job* harian/mingguan yang mengirimkan pesan teks rekap performa ke Telegram Super Admin.

## 6. Arsitektur Database (Panduan Migrasi AI)
Buat struktur relasional yang kokoh. (*Developer Note: Gunakan tipe data Decimal untuk uang, bukan Integer*).

* **`users`**: id, name, email, password, role.
* **`rooms`**: id, name, description, price, capacity, status (Available, Cleaning, Maintenance).
* **`bank_accounts`**: id, bank_name, account_number, account_name, is_active.
* **`vouchers`**: id, code, type (percent/fixed), value, is_active.
* **`bookings`**:
    * id, booking_code, guest_name, guest_phone, acquisition_source.
    * room_id, check_in_date, check_out_date.
    * total_room_price, total_addons_price, late_fee, discount_amount, grand_total.
    * paid_amount, balance_due.
    * payment_status (Pending, DP, Lunas, Cancelled).
    * booking_status (Booked, In-House, Completed, No-Show).
* **`booking_addons`**: id, booking_id, item_name, type (food/extrabed), qty, price, subtotal, status (Pending, Paid, Delivered).
* **`settings`**: id, key_name, value (Contoh: `min_dp_percent` = 50).

## 7. Instruksi Eksekusi & Keamanan untuk AI Developer
1.  **Validasi Middleware:** Lindungi semua *route* modifikasi keuangan. Contoh: `Route::post('/validate-payment')->middleware('role:super_admin');`
2.  **Date Filtering Logic:** Gunakan logika SQL `WHERE (check_in_date < ? AND check_out_date > ?)` untuk mengecek kamar yang tumpang tindih (*overlap*) saat tamu mencari kamar.
3.  **UI/UX:** Desain *mobile-first*. Form harus mudah ditekan di layar HP. Gunakan warna *earth-tone* (hijau sage, putih, abu-abu gelap) untuk kesan elegan.
4.  **Cron Jobs cPanel:** Siapkan *script console command* di Laravel (`php artisan schedule:run`) untuk pengiriman laporan Telegram mingguan dan *backup database* otomatis.
